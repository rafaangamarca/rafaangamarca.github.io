<h3>1. OBJETIVO DEL SPRINT</h3>
<p>
  Completar el desarrollo integral de la plataforma Fit Push, asegurando la funcionalidad técnica de todas las herramientas de entrenamiento, nutrición y asistencia interactiva, así como la optimización técnica para motores de búsqueda (SEO).
</p>

<h3>2. BACKLOG DEL SPRINT (HISTORIAS DE USUARIO Y TAREAS)</h3>
<p>Durante este ciclo de desarrollo se han completado las siguientes unidades funcionales:</p>
<ul>
  <li><strong>Interfaz Principal (Index):</strong> Diseño y maquetación del ecosistema visual bajo una estética de alto contraste.</li>
  <li><strong>Generador de Rutinas:</strong> Implementación de lógica programática para la creación de planes de entrenamiento personalizados.</li>
  <li><strong>Módulo de Suplementación:</strong> Desarrollo de una base de conocimiento basada en evidencia científica.</li>
  <li><strong>Calculadora Nutricional (Macros):</strong> Integración de algoritmos basados en la fórmula Mifflin-St Jeor.</li>
  <li><strong>Módulo de Tendencias:</strong> Integración de Google Trends para el análisis de datos de búsqueda en tiempo real.</li>
  <li><strong>Asistencia Inteligente (Chatbot):</strong> Implementación de un agente conversacional vía SDK de Typebot.</li>
  <li><strong>Sistema de Testeo Técnico:</strong> Creación de un panel de verificación SEO y rendimiento.</li>
</ul>

<h3>3. PROCESO DE EJECUCIÓN Y HITOS ALCANZADOS</h3>

<p><strong>Fase 1: Arquitectura Base y Biblioteca</strong></p>
<p>Se estableció el archivo index.html como núcleo del proyecto, definiendo las variables de CSS (Verde Neón y Dark Mode) y la biblioteca de ejercicios con filtrado dinámico.</p>

<p><strong>Fase 2: Lógica de Entrenamiento y Nutrición</strong></p>
<p>Se desarrollaron las páginas rutina.html y macros.html. En la primera se implementó un sistema de gestión de tiempo con cronómetros de descanso, mientras que en la segunda se programó la lógica de reparto de macronutrientes.</p>

<p><strong>Fase 3: Contenido Científico y Datos Externos</strong></p>
<p>Se finalizó la sección suplementos.html asegurando la legibilidad técnica. Posteriormente, se integró el widget de Google Trends en el inicio para dotar a la web de datos dinámicos externos.</p>

<p><strong>Fase 4: Interactividad y Calidad Técnica</strong></p>
<p>Se integró el chatbot mediante la carga asíncrona de scripts para soporte al usuario. El hito final consistió en el despliegue de un panel de testeo SEO para monitorizar etiquetas meta, archivos sitemap.xml y robots.txt.</p>
